# DEV-03 — دليل إعداد بيئة التطوير

**منصة تجارة الجملة الإلكترونية B2B / B2C**
مستند المرحلة الرابعة (التطوير) · الإصدار 1.0

هذا الملف يُحفظ في جذر المستودع باسم `README.md` ويُحدَّث مع كل تغيير في الإعداد. الهدف: أن يشغّل أي مساهم جديد (أو أداة ذكاء اصطناعي) المشروع كاملاً خلال أقل من 15 دقيقة.

## التشغيل السريع (Backend)

```bash
git clone <repo-url> && cd wholesale-platform
cp .env.example .env                 # عدّل القيم المحلية عند الحاجة
docker compose up -d                 # PostgreSQL + Redis + Elasticsearch + RabbitMQ
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
alembic upgrade head                 # تطبيق الهجرات (لا create_all)
python -m app.seed                   # بيانات بذور للتطوير
uvicorn app.main:app --reload        # الخادم على http://localhost:8000
```

توثيق API التفاعلي بعد التشغيل: `http://localhost:8000/docs` (مولّد من عقد DES-03).

## القواعد الذهبية قبل أول commit

هذه ملخص تنفيذي من **DEV-01**، والالتزام بها شرط لقبول أي كود:

1. **المال `Decimal` دائماً** — لا `Float` في أي قيمة نقدية.
2. **لا أسرار في الكود** — كل شيء في `.env` (غير متعقّب في Git).
3. **لا رمز OTP أو بيانات حساسة في أي استجابة API** — ولو في التطوير.
4. **خصم المخزون ذري** داخل معاملة، لا فحص ثم خصم منفصلين.
5. **التوكنات في كوكي HttpOnly** — لا `localStorage`.
6. **كل تغيير مخطط عبر Alembic** — يعمل صعوداً ونزولاً.
7. **الكود المولّد بالذكاء الاصطناعي مقترح لا حقيقة** — يمر ببوابة المراجعة (DEV-01 §6) قبل الدمج.

## بنية المستودع

```
wholesale-platform/
├── backend/
│   ├── app/
│   │   ├── api/          # طبقة HTTP فقط (routers)
│   │   ├── services/     # منطق الأعمال
│   │   ├── repositories/ # الوصول للبيانات
│   │   ├── schemas/      # عقود Pydantic
│   │   ├── models/       # جداول SQLAlchemy
│   │   └── main.py
│   ├── alembic/          # الهجرات
│   └── tests/
├── apps/
│   ├── buyer/            # تطبيق المشتري (Flutter)
│   ├── supplier/         # بوابة المورد
│   └── driver/           # تطبيق السائق
├── admin/                # لوحة الإدارة (ويب)
├── shared/               # نماذج وثوابت مشتركة
├── docker-compose.yml
└── .env.example
```

**الحدود بين الطبقات ملزمة** (تفصيلها في DEV-01 §2): الـrouter لا يحوي منطقاً، والمنطق في الخدمات، والبيانات في المستودعات. هذا الفصل هو ما يجعل المشروع قابلاً للتوسع، وغيابه كان أكبر مشكلة في النموذج الأولي.
